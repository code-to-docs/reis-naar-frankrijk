UI NORMPROFIEL — Reis naar Frankrijk
Versie: 2025-01-09

1. Visie & Principes
Waarom dit document?
Dit normprofiel definieert hoe onze reisplanner eruitziet, aanvoelt en zich gedraagt. Het is de bron van waarheid voor elke designbeslissing — van een enkele knop tot een complete pagina.

Ontwerpvisie
Een reisplanner die aanvoelt als een betrouwbare metgezel: helder wanneer je informatie zoekt, stil wanneer je droomt, en behulpzaam wanneer je plant.

Vijf ontwerpprincipes
Principe	Betekenis	In de praktijk
1. Lucht	Genereuze ruimte laat content ademen. Witruimte is geen leegte maar rust.	Minimaal 16px padding rond content. Nooit elementen tegen elkaar "plakken".
2. Helderheid	Elk element communiceert direct zijn functie. Geen decoratie zonder doel.	Eén primaire actie per scherm. Labels zijn concreet ("Opslaan", niet "OK").
3. Zachtheid	Afgeronde vormen en vloeiende overgangen creëren warmte.	Consistente border-radius. Geen harde cuts, maar eased transitions.
4. Vertrouwen	Voorspelbare patronen. De app gedraagt zich zoals verwacht.	Dezelfde interactie werkt overal hetzelfde. Geen verrassingen.
5. Vreugde	Subtiele details die ontdekt worden, niet opgedrongen.	Micro-animaties bij succesacties. Thoughtful empty states.
2. Visuele Identiteit
Sfeer & emotie
Avontuurlijk maar betrouwbaar — de opwinding van reizen, de zekerheid van goede voorbereiding
Modern maar warm — clean interfaces met menselijke touches
Licht en uitnodigend — geen zware, donkere esthetiek tenzij in dark mode
Referentie-apps (voor gevoel, niet kopiëren)
Apple Maps — ruimtelijke hiërarchie, kaartintegratie
Flighty — data-dense maar leesbaar, mooie typografie
Airbnb — content-forward, uitnodigende imagery
Apple Journal — zachte kleuren, persoonlijke sfeer
Fotografie & imagery (indien gebruikt)
Natuurlijk licht, niet overbewerkt
Mensen mogen voorkomen maar zijn niet centraal
Franse landschappen, architectuur, food als sfeerbeeld
Nooit stockfoto-achtig of generiek

4. Typografie
Ontwerpfilosofie typografie
Eén fontfamilie voor consistentie en snelheid
Hiërarchie via grootte en gewicht, niet via kleur of decoratie
Leesbare line-heights: ruimer voor bodytekst, strakker voor koppen
Voldoende contrast met achtergrond in alle modi
Fontfamilie
css
--font-sans: "Inter", -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif;
--font-mono: "SF Mono", "Fira Code", "Consolas", monospace;
Waarom Inter? — Uitstekende leesbaarheid op scherm, variabele font support, gratis, en visueel verwant aan SF Pro.

Typografische schaal
Gebaseerd op een 1.2 ratio (minor third) met 16px als basis.

Typografieregels
Regel	Toelichting
Geen italic voor lange tekst	Alleen voor wetenschappelijke namen, boektitels, of korte nadruk
UPPERCASE spaarzaam	Alleen voor korte labels, badges, of navigatie-items. Nooit voor zinnen.
Max leeslengte 65–75 karakters	Gebruik max-width: 65ch voor leestekst
Geen underline tenzij link	Underline = klikbaar. Gebruik anders font-weight of kleur.

8. Motion & Animatie
Ontwerpfilosofie motion
Animatie ondersteunt begrip, niet decoratie
Snelle feedback voor directe interacties
Langzamere, soepelere transities voor grote veranderingen
Respecteer prefers-reduced-motion
Durations
css
--duration-instant: 0ms;
--duration-fast: 100ms;      /* Hover, focus states */
--duration-normal: 200ms;    /* Default transitions */
--duration-slow: 300ms;      /* Modals, page transitions */
--duration-slower: 500ms;    /* Complex animations */
Easing curves
css
/* Standaard — voor de meeste UI-transities */
--ease-default: cubic-bezier(0.25, 0.1, 0.25, 1);

/* In — elementen die binnenkomen */
--ease-in: cubic-bezier(0.42, 0, 1, 1);

/* Out — elementen die verdwijnen */
--ease-out: cubic-bezier(0, 0, 0.58, 1);

/* In-Out — positieveranderingen */
--ease-in-out: cubic-bezier(0.42, 0, 0.58, 1);

/* Expressive — voor "spring-like" feel */
--ease-spring: cubic-bezier(0.175, 0.885, 0.32, 1.275);

/* Smooth — natuurlijke vertraging */
--ease-smooth: cubic-bezier(0.4, 0, 0.2, 1);
Transition presets
css
--transition-colors: color, background-color, border-color, fill, stroke;

.transition-transform {
  transition-property: var(--transition-transform);

9. Iconografie
Ontwerpfilosofie iconen
Iconen ondersteunen tekst, vervangen deze zelden volledig
Consistente stijl: outline OF filled, niet gemixed
Voldoende grootte voor herkenbaarheid
Kleur volgt tekstkleur tenzij semantisch
Icoonspecificaties
Property	Waarde
Stijl	Rounded outline (stroke-based)
Stroke width	1.5px - 2px
Corners	Rounded
Minimum grootte	16x16px
Touch-icoon grootte	24x24px in 44x44px touch target
Standaard grootte	20x20px of 24x24px
Icoon sizes
css
--icon-xs: 12px;
--icon-sm: 16px;
--icon-md: 20px;
--icon-base: 24px;
--icon-lg: 28px;
--icon-xl: 32px;
--icon-2xl: 40px;
Aanbevolen icon libraries
Lucide (primair) — Clean, consistent, open source
Heroicons — Alternatief, iets "voller"
SF Symbols stijl — Als je maximaal Apple-gevoel wilt
Icoonregels
Regel	Toelichting
Altijd met label	Standalone iconen alleen voor universele acties (close, menu, search)
Consistente set	Mix geen icon libraries in dezelfde view
Optische uitlijning	Sommige iconen (bijv. play) hebben visuele correctie nodig
Touch area	Icon mag klein zijn, touch target niet

Contrast
Element	Minimum ratio
Normale tekst (<18px)	4.5:1
Grote tekst (≥18px bold, ≥24px regular)	3:1
UI componenten & graphics	3:1
Focus indicators	3:1 met achtergrond

14. Review Checklist
Voor elke UI-wijziging
Tokens & Consistentie
 Gebruikt alle tekst bestaande typografie tokens?
 Zijn kleuren semantische tokens (geen hex/rgb direct)?
 Is spacing uit het spacing systeem?
 Zijn border-radius waarden van het radius systeem?
Hierarchie & Duidelijkheid
 Is er maximaal één primaire CTA per viewport?
 Is de visuele hierarchie duidelijk (titel → content → actie)?
 Zijn labels concreet en actiegericht?
Interactie
 Zijn touch targets minimaal 44×44px (36px voor compact)?
 Hebben alle interactieve elementen hover/focus/active states?
 Is focus zichtbaar en logisch geordend?
Toegankelijkheid
 Is contrast minimaal 4.5:1 (tekst) / 3:1 (UI)?
 Werkt de component met keyboard only?
 Zijn decoratieve elementen aria-hidden?
 Heeft elk interactief element een accessible name?
Responsive
 Werkt het op 320px breedte (kleine mobiel)?
 Geen horizontale scroll op normale content?
 Schalen touch targets mee (niet te klein op desktop)?
Motion
 Zijn animaties subtiel en purposeful?
 Respecteert prefers-reduced-motion?
Dark Mode
 Zijn alle kleuren via tokens (automatische dark mode)?
 Is contrast ook in dark mode voldoende?
 Geen "flashing" bij mode switch?
